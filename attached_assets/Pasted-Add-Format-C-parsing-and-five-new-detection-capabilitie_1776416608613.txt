Add Format C parsing and five new detection capabilities for the Federal Appropriation Bill (2,790 pages). This is the highest priority update.
PART 1 — Format C Parser (parser.py)
The Federal Appropriation Bill has two sections:

Pages 1–9: MDA summary (already partially handled — keep as is)
Pages 10–2790: Project-level detail — this is Format C

Format C page structure:
0234004001     FEDERAL ROAD MAINTENANCE AGENCY        ← MDA section header

CODE              PROJECT NAME                    TYPE      AMOUNT
ERGP12234385   Maintenance of Zaria-Pambeguwa     ONGOING   14,700,000,000
               Road, Kaduna State
ERGP12234386   Roads Rehabilitation in Selected   NEW       7,000,000,000
               Locations in North West (Multiple
               Locations)
Detection: Format C is active when format_b_detected = False AND the page contains a pattern matching [A-Z]{2,4}\d{8,12} (ERGP codes) AND contains "ONGOING" or "NEW".
Parsing rules for Format C:

Track current MDA by detecting 10-digit section headers ^\d{10}\s+[A-Z] — store as current_mda_code and current_mda_name
For each project line, extract:

project_code: token matching [A-Z]{2,4}\d{8,12}
description: everything between the code and TYPE keyword (ONGOING/NEW), including continuation lines
status: ONGOING or NEW
amount: last number on the line
ministry: current_mda_name
expenditure_code: detect from lines starting with 21 (Personnel), 22 (Overhead), 23 (Capital) — only extract Capital (23) items for red-flag analysis, skip Personnel and Overhead lines


Multi-line descriptions: if a line has no ERGP code and no amount, append it to the previous row's description
Set is_mda_level = False for all Format C rows
Skip lines that are pure category headers (start with 2-digit expenditure codes like 21, 22, 23, 2101, 2202 etc.)

PART 2 — Five New Flag Types (flags.py)
Flag A — VAGUE LOCATION (new flag type)
Fire when description contains any of: "selected locations", "multiple locations", "multiple lots", "various locations", "selected states", "various states", "nationwide", "across the country"

Severity: HIGH if amount > ₦1B, MEDIUM otherwise
Title: "Non-Traceable Location"
Explanation: "Project description uses vague location language ('[matched phrase]'). Without specific locations, implementation cannot be tracked or verified."

Flag B — BUDGET SPLITTING (new flag type)
Detect when 3 or more items share: same MDA + similar description (85%+ match) + amounts within 5% of each other

Group these as a "split cluster"
Severity: HIGH
Title: "Suspected Budget Splitting"
Explanation: "Found [N] line items under [MDA] with near-identical descriptions and amounts (₦X each). This pattern is consistent with project splitting to avoid oversight thresholds."

Flag D — ENHANCED DUPLICATE with splitting detection
Extend the existing duplicate cluster flag: if duplicate cluster members all have amounts within 10% of each other AND they are all under the same MDA, upgrade severity to HIGH and add note "Identical amounts suggest systematic splitting rather than coincidental duplication."
